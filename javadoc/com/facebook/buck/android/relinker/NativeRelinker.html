com.facebook.buck.android.relinker

Class NativeRelinker

  • java.lang.Object
    • com.facebook.buck.android.relinker.NativeRelinker

  • public class NativeRelinker
extends Object
    When linking shared libraries, by default, all symbols are exported from the library. In a particular application, though, many of those symbols may never be used. Ideally, in each apk, each shared library would only export the minimal set of symbols that are used by other libraries in the apk. This would allow the linker to remove any dead code within the library (the linker can strip all code that is unreachable from the set of exported symbols).

    The native relinker tries to remedy the situation. When enabled for an apk, the native relinker will take the set of libraries in the apk and relink them in reverse order telling the linker to only export those symbols that are referenced by a higher library.
