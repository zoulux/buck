Interface BuildableContext

  • All Known Implementing Classes:
    DefaultBuildableContext

    public interface BuildableContext
    Context object that is specific to an individual BuildRule. This differs from BuildContext in that a BuildContext is a context that is shared by all BuildRules whereas a new BuildableContext is created for each call to BuildRule.getBuildSteps(BuildContext, BuildableContext).

      • addMetadata

        void addMetadata(String key,
               String value)
        When building a BuildRule, any metadata about the output files for the rule should be recorded via this method. This ensures it will be stored in the ArtifactCache along with the output.

        If a value for the specified key has already been set, an exception will be thrown. (If we discover a good reason for this to be allowed later, we can always relax this constraint.)
