      • getWantsPackagePrefix

        public boolean getWantsPackagePrefix()
        Used to make IntelliJ ignore the package name->folder structure convention and assume the given package prefix. An example of a scenario this makes possible to achieve is having java/src/Foo.java declare the package "org.bar" (instead of having the path to the file be java/org/bar/Foo.java). The main effect of this is the elimination of IntelliJ warnings about incorrect package prefixes and having it use the correct package when creating new files.
        Returns:
        whether to generate package prefix for this folder.
